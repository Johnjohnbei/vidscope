# S01: MCP server foundation with 5 read-only tools

**Goal:** Ship the MCP server package: install the mcp SDK, add a new `vidscope.mcp` interface layer wrapping the 5 existing M001 use cases (ingest, search, get_video, list_videos, get_status) as FastMCP tools, add a CLI `vidscope mcp serve` entry point, extend the import-linter contract with an `mcp` layer, and validate via unit tests (direct handler calls) plus a subprocess integration test that spawns the server and lists tools via JSON-RPC.
**Demo:** After this: Starting `vidscope mcp serve` exposes 5 tools (ingest, search, get_video, list_videos, get_status) that an MCP client can call via JSON-RPC. A subprocess integration test confirms the server responds to list_tools.
