# S04: Frame extraction brick (ffmpeg)

**Goal:** Add frame extraction to the pipeline: an FfmpegFrameExtractor adapter that shells out to ffmpeg, a FramesStage that orchestrates FrameExtractor + MediaStorage + FrameRepository, container wiring, and integration tests proving real frame extraction works on real downloaded media. Default strategy: keyframes plus ~1 frame per 5 seconds, capped at 30 frames per video (per D016/R003 notes).
**Demo:** After this: `vidscope add <url>` extracts keyframes plus a fixed-rate sample from the downloaded media, stores the image files on disk, and records their paths and timestamps in the `frames` table.

## Tasks
- [x] **T01: Shipped FfmpegFrameExtractor: subprocess wrapper with shutil.which preflight, configurable fps, max_frames cap, typed errors for all 6 failure modes — 9 stubbed unit tests, ffmpeg subprocess never invoked.** — Create src/vidscope/adapters/ffmpeg/__init__.py and src/vidscope/adapters/ffmpeg/frame_extractor.py. FfmpegFrameExtractor implements FrameExtractor port. extract_frames(media_path, output_dir, max_frames=30): (1) verify ffmpeg binary is on PATH via shutil.which, raise FrameExtractionError(retryable=False) with install instructions if not. (2) Build an ffmpeg command that extracts frames at a fixed FPS rate (e.g. 0.2 fps = 1 frame per 5 seconds) with output template output_dir/frame_%04d.jpg. (3) Run subprocess with timeout 60s, capture stderr. (4) On non-zero exit, raise FrameExtractionError with the stderr tail. (5) Glob the output_dir for *.jpg, sort by name, cap at max_frames, build a list of Frame entities with timestamp_ms computed from the frame index and the configured fps. Return the list. Tests stub subprocess.run + the glob with monkeypatch — zero real ffmpeg invocation in unit tests.
