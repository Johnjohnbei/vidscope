# S07: Cookie-based authentication for Instagram (and other gated content)

**Goal:** Add optional cookie-based authentication to YtdlpDownloader so the user can ingest gated content (primarily Instagram public Reels which now require auth, secondarily age-gated YouTube and private TikTok). The cookies file path is resolved from VIDSCOPE_COOKIES_FILE env var with a sensible default under data_dir. The change is purely additive: existing public-content workflows continue to work without cookies. Validates R001 against Instagram (the user's #1 priority platform per D027) and promotes R025 from deferred to validated.
**Demo:** After this: `vidscope add <instagram-reel-url>` succeeds end-to-end on a public Reel after the user supplies a `cookies.txt` file (or sets `VIDSCOPE_COOKIES_FILE`); R025 promoted from deferred to validated; the integration test for Instagram flips from xfail to passing.
